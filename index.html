Web Security

How would you advise a potential client to secure his/her website from web security attacks? Note: the word count for your answer should not be greater than 250 words

To create and maintain website security, one must focus on the entire website. That is, both on the server, the application and the client.

Cross-Site Scripting (XSS) has been the most common type of security threat. The best way to protect against XSS attacks is to avoid elements such as <script>, <object>, <embed> and <link> in the code. Web frameworks often automatically sanitize user input from HTML forms.

SQL injection is another type of security threat. It occurs when you have the opportunity to change the SQL query to a database by writing code as input in a form in the browser. To avoid this, you can put a backslash before the ' character, so that input from the browser is treated as a text string, and not as code.

Cross-Site Request Forgery (CSRF) is a threat where the attacker spreads malicious code to many recipients via email. The code is executed when the recipient presses the Send button while logged into a website, and the individual's login details to the website are already stored in the browser, making this possible. If the POST request sends a secret code to each user on the site, which they must confirm, the malicious code will not work anyway.

Strong passwords and two-factor authentication is recommended.

There are several other threats to consider, but the most important is to never trust data from the browser, such as URL parameters for GET and POST requests, HTTP headers, cookies, and user-uploaded files. Always assume the worst.
